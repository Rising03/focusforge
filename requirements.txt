## Installation Instructions

### Quick Setup (Recommended)

#### Windows:
```cmd
setup.bat
```

#### Linux/macOS:
```bash
chmod +x setup.sh
./setup.sh
```

### Manual Setup

### 1. Clone Repository
```bash
git clone <repository-url>
cd student-discipline-system
```

### 2. Install All Dependencies
```bash
# Automated installation (recommended)
npm run install:all

# OR Manual installation
npm install                           # Root dependencies
cd frontend && npm install --legacy-peer-deps  # Frontend dependencies
cd ../backend && npm install          # Backend dependencies
cd ..                                # Return to root
```

### 3. Setup Database
```bash
# Start PostgreSQL and PgAdmin
cd database
docker-compose up -d
cd ..
```

### 4. Environment Configuration
```bash
# Backend environment (already configured)
# File: backend/.env
# Contains database connection, JWT secrets, API keys
```

### 5. Start Development Servers
```bash
# Start both frontend and backend
npm run dev
```

## Service URLs
- Frontend: http://localhost:5173
- Backend API: http://localhost:3001
- Database: localhost:5432
- PgAdmin: http://localhost:5050 (admin@example.com / admin)

## Build Commands
```bash
# Build for production
npm run build

# Run tests
npm run test

# Individual services
npm run dev:frontend    # Frontend only
npm run dev:backend     # Backend only
npm run build:frontend  # Build frontend
npm run build:backend   # Build backend
npm run test:frontend   # Test frontend
npm run test:backend    # Test backend
```

## Troubleshooting

### Common Issues
1. Port conflicts: Ensure ports 3001, 5173, 5432, 5050 are available
2. Docker not running: Start Docker Desktop
3. Node version: Use Node.js v18 or higher
4. Dependency conflicts: Use --legacy-peer-deps flag for frontend

### Reset Instructions
```bash
# Clean install
rm -rf node_modules frontend/node_modules backend/node_modules
rm package-lock.json frontend/package-lock.json backend/package-lock.json
npm run install:all
```
